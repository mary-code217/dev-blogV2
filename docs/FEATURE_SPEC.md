# 기능 상세 정의서

## 1. 홈 페이지

| 항목 | 내용 |
|---|---|
| 경로 | `/` |
| 설명 | 블로그 메인 페이지, 최신 글 미리보기 |

### 동작
- 최신 글 **6개** 표시 (설정값으로 조정 가능)
- 각 글 카드: 제목, 설명, 날짜, 카테고리
- 하단 "전체 글 보기" 버튼 → `/blog/` 이동
- `draft: true` 글은 제외

---

## 2. 글 목록 페이지

| 항목 | 내용 |
|---|---|
| 경로 | `/blog/` |
| 설명 | 전체 글 목록, 필터링 및 검색 |

### 정렬
- 기본: 최신순 (날짜 내림차순)

### 카테고리 필터
- 글 목록 상단에 카테고리 탭/버튼 표시
- 클릭 시 해당 카테고리 글만 필터링
- "전체" 옵션으로 필터 초기화

### 태그 필터
- 글 카드 내 태그 클릭 시 해당 태그로 필터링
- 또는 별도 태그 클라우드 제공
- 카테고리 필터와 **조합 가능** (카테고리 + 태그 동시 필터)

### 검색
- Pagefind 기반 클라이언트 사이드 검색
- 검색 대상: 제목, 본문 내용, 태그
- 검색창은 글 목록 상단에 배치
- 빌드 시 검색 인덱스 자동 생성

### 페이지네이션
- **"더보기" 버튼** 방식
- 초기 로드: 10개
- 버튼 클릭 시 10개씩 추가 로드
- 더 이상 글이 없으면 버튼 숨김

## 3. 글 상세 페이지

| 항목 | 내용 |
|---|---|
| 경로 | `/blog/{slug}/` |
| 설명 | 개별 글 내용 렌더링 |

### 헤더 영역
- 글 제목
- 발행일 (`date`)
- 카테고리 (클릭 시 해당 카테고리 필터로 이동)
- 태그 목록 (클릭 시 해당 태그 필터로 이동)

### 본문 영역
- Markdown → HTML 렌더링
- 코드 블록: Syntax Highlighting + 복사 버튼 + 언어 표시
- 이미지: 반응형 처리

### 목차 (Table of Contents)
- 글 본문의 `h2`, `h3` 제목을 자동 추출하여 목차 생성
- **데스크톱**: 우측에 플로팅 목차 (스크롤 따라다님)
- **모바일**: 글 상단에 접이식 목차
- 현재 읽고 있는 섹션 하이라이팅 (스크롤 스파이)
- 목차 항목 클릭 시 해당 섹션으로 스무스 스크롤
- 제목에 앵커 링크 자동 생성 (URL 공유 가능)

### 하단 네비게이션
- 이전 글 / 다음 글 링크
- 날짜 순서 기준

## 4. 검색

| 항목 | 내용 |
|---|---|
| 도구 | Pagefind |
| 방식 | 클라이언트 사이드 (빌드 타임 인덱싱) |

### 동작
1. `npm run build` 시 Pagefind가 전체 글을 인덱싱
2. 검색 UI는 글 목록 페이지에 통합
3. 사용자 입력 시 실시간 필터링 (디바운스 적용)
4. 검색 결과: 제목, 발췌문, 하이라이팅 표시

### 검색 대상
- 글 제목 (`title`)
- 글 본문 (Markdown 내용)
- 태그 (`tags`)

---

## 5. 카테고리 & 태그

### 카테고리
| 항목 | 내용 |
|---|---|
| 관계 | 글당 **1개** (필수) |
| 용도 | 주요 분류 기준 |

초기 카테고리 (예시):
- `Development` — 개발 관련
- `DevOps` — 인프라, 배포
- `TIL` — Today I Learned
- `Retrospective` — 회고

> 카테고리는 Frontmatter의 `category` 필드에서 자동 수집 (별도 설정 파일 불필요)

### 태그
| 항목 | 내용 |
|---|---|
| 관계 | 글당 **N개** (선택) |
| 용도 | 세부 주제 분류 |

- Frontmatter의 `tags` 배열에서 자동 수집
- 예: `["Astro", "JavaScript", "CSS"]`
- 글 상세 및 글 목록에서 태그 클릭 시 필터링

## 6. 다크/라이트 모드

### 동작
1. 첫 방문 시 **시스템 설정** 반영 (`prefers-color-scheme`)
2. 토글 버튼 클릭 시 모드 전환
3. 선택한 모드는 **localStorage**에 저장
4. 재방문 시 저장된 설정 우선 적용

### 구현
- `<html>` 태그에 `data-theme="dark"` 또는 `data-theme="light"` 속성
- CSS 변수로 색상 관리
- 토글 버튼: 사이드바(데스크톱) / 헤더(모바일)에 배치
- 깜빡임 방지: `<head>`에 인라인 스크립트로 초기 테마 설정

## 7. 코드 블록

### Syntax Highlighting
- **Shiki** 사용 (Astro 기본 내장)
- 라이트/다크 테마별 다른 코드 테마 적용
  - 라이트: `github-light`
  - 다크: `github-dark`

### 복사 버튼
- 각 코드 블록 우측 상단에 복사 아이콘 버튼
- 클릭 시 코드 내용 클립보드 복사
- 복사 완료 시 체크 아이콘으로 피드백 (1.5초 후 복원)

### 언어 표시
- 코드 블록 상단에 언어명 표시 (예: `javascript`, `python`)
- Markdown의 ` ```js ` 구문에서 자동 추출

## 8. 사이드바 네비게이션

### 데스크톱 (900px 이상)
- 좌측 고정 사이드바
- 구성:
  - 블로그 타이틀 (**MaryDev**) → 홈 링크
  - 메뉴: 홈, 글 목록
  - 다크/라이트 모드 토글
- 현재 페이지 메뉴 하이라이팅

### 모바일 (900px 미만)
- 사이드바 → 상단 헤더로 전환
- 헤더: 타이틀 + 메뉴 항목 + 다크모드 토글
- 필요 시 햄버거 메뉴로 축소

## 9. RSS 피드

| 항목 | 내용 |
|---|---|
| 경로 | `/rss.xml` |
| 라이브러리 | `@astrojs/rss` |

### 동작
- 전체 공개 글 포함 (`draft: false`)
- 최신순 정렬
- 포함 정보: 제목, 설명, 발행일, 링크
- `<head>`에 RSS 자동 발견 링크 추가

## 10. SEO

### Open Graph
- 각 페이지별 OG 메타 태그 자동 생성
  - `og:title` — 글 제목 또는 사이트 제목
  - `og:description` — 글 설명
  - `og:url` — canonical URL
  - `og:type` — `article` (글) 또는 `website` (홈)

### 사이트맵
- `@astrojs/sitemap` 사용
- 빌드 시 `/sitemap-index.xml` 자동 생성
- `draft: true` 글 제외

### 기타
- canonical URL 설정
- HTML lang 속성 (`ko`)
- 시맨틱 HTML 태그 사용 (`<article>`, `<nav>`, `<main>`)
